# 2. Introduction

This document describes the testing approach, scope, objectives, resources, and deliverables for manual testing of the SauceDemo web application.

The objective is to validate the application's functionality, usability, and reliability before release by identifying defects and ensuring that business requirements are met.

---

# 3. Objectives

- Verify all major application functionalities.
- Ensure business requirements are satisfied.
- Identify functional defects.
- Validate user workflows.
- Minimize production defects.

---

# 4. Scope

### In Scope

- UI Validation
- Cross-browser Functional Testing

### Out of Scope

- Payment Gateway

- Security Testing
- Database Validation

# 7. Entry Criteria

- Application deployed successfully.
- Requirements available.
- Test environment accessible.
- Test data prepared.

---

# 8. Exit Criteria

- All critical test cases executed.
- No Critical or High severity defects remain open.
- Test Summary Report completed.
- Client approval received.

---

# 9. Deliverables

- Test Plan
- Test Scenarios
- Test Cases
- Bug Reports
- Requirement Traceability Matrix (RTM)
- Test Summary Report

---

# 10. Risks

- Environment downtime
- Requirement changes
- Limited test data
- Browser compatibility issues

---

# 11. Assumptions

- Stable test environment.
- Test accounts remain active.
- Requirements are finalized before execution.
